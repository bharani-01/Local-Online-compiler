Online Python Compiler
This project provides a web-based environment for writing and executing Python code, demonstrating two distinct architectures: Server-Side Execution (using Flask) and Client-Side Execution (using Pyodide/WebAssembly).

🚀 Key Features
Dual Architecture: Supports both fast, client-side execution and robust, server-side execution.

Syntax Highlighting: Integrated CodeMirror editor for a professional coding experience.

Input Handling: Supports providing input (stdin) to the Python script.

Persistence: Code and input are saved to the browser's local storage.

Single-File Frontend: Both frontends (index.html and test.html) are self-contained with embedded HTML, CSS (Tailwind), and JavaScript.

💻 Architecture Overview
1. Server-Side Execution (Recommended for heavier tasks)
Files: app.py, index.html

Backend: Python Flask application (app.py).

Execution: The frontend (index.html) sends the code and pre-defined input to the /run endpoint. The Flask backend executes the code in a sandboxed subprocess on the server, capturing the output and error streams before returning the result.

How to Access: Run the Flask server (see Getting Started below) and navigate to the application's URL.

2. Client-Side Execution (Recommended for instant feedback)
File: test.html

Execution: This version uses Pyodide (CPython compiled to WebAssembly) to run the Python interpreter directly within the user's browser. No backend server is required for code execution.

Input: It supports interactive input prompts by setting up a custom stdin handler within Pyodide.

How to Access: Simply open test.html in your web browser.

⚙️ Getting Started (Server-Side)
To run the Flask application for the server-side compiler:

Prerequisites
You need Python 3.x installed.

1. Setup Environment
Create and activate a virtual environment (recommended):

python3 -m venv venv
source venv/bin/activate  # On macOS/Linux
.\venv\Scripts\activate   # On Windows


2. Install Dependencies
Install the necessary Python packages from the requirements.txt file:

pip install -r requirements.txt


3. Run the Server
Start the Flask application:

python app.py


The application will typically be available at http://127.0.0.1:5001/.

4. Access the Compiler
Server-Side Compiler: Open your browser to the URL provided by the Flask output (usually http://127.0.0.1:5001/).

Client-Side Compiler: Open the test.html file directly in your browser.
